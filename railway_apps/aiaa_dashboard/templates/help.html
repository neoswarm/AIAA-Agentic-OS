Getting Started

1
Set up your API key
Go to Settings and add your OpenRouter API key. This powers all AI content generation.
2
Add a client
Visit Clients to set up your first client profile. This personalizes all content created for them.
3
Run your first skill
Browse available skills from the Workflows page and run one to see how it works.
4
Review your output
Check Execution History to see your results. You can copy, download, or send to Google Docs.

Glossary

Skill
A pre-built task that generates a specific type of output, like a blog post, email sequence, or research report.
API Key
A unique code that connects Agentic OS to external services like AI models (OpenRouter) or research tools (Perplexity).
Client Profile
A saved set of information about one of your clients, used to personalize content and maintain consistency.
Execution
A single run of a skill. Each execution produces an output and is logged in your history.
Webhook
A way for external services to trigger actions in Agentic OS automatically, like starting a skill when a new lead comes in.
Brand Voice
The tone and style used in content. Can be casual, professional, technical, friendly, or authoritative.
Deliverable
The finished output from a skill, like a completed blog post, email draft, or research report.

Troubleshooting

"API key not working" or "Unauthorized" errors
Fix: Go to Settings and re-enter your API key. Make sure you copied the full key including the prefix (e.g., sk-or-). If it still fails, generate a new key from your provider.
Skill is taking too long or seems stuck
Fix: Some skills take 2-5 minutes, especially research-heavy ones. If it's been more than 10 minutes, try cancelling and running again. Check that your API key has sufficient credits.
Output quality is lower than expected
Fix: Make sure you've filled in as many input fields as possible. Skills produce better results with detailed inputs. Also check that you have a client profile set up with target audience and brand voice configured.
Google Docs delivery isn't working
Fix: Google Docs integration requires a credentials file on the server. Check Settings to see if Google Docs shows as connected. Your outputs are always saved locally even if delivery fails.
Slack notifications aren't coming through
Fix: Verify your Slack Webhook URL in Settings. Use the Test button to send a test message. Make sure the webhook is still active in your Slack workspace settings.
Page shows an error after logging in
Fix: Try clearing your browser cache and logging in again. If the issue persists, check that the dashboard service is running correctly in Railway.

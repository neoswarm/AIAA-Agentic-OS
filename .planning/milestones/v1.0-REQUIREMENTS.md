## v1 Requirements

### Input Validation (VAL)
- [x] **VAL-01**: All skill execution form fields validate required inputs before submission with inline error messages
- [x] **VAL-02**: Client form validates required fields (name) and formats (website URL) with real-time feedback
- [x] **VAL-03**: Settings page validates API key format before save attempt (length, prefix patterns)
- [x] **VAL-04**: Server-side validation on all API v2 endpoints returns structured error JSON with field-level messages
- [x] **VAL-05**: Search inputs sanitize and debounce to prevent excessive API calls

### Error Handling (ERR)
- [x] **ERR-01**: All API calls in JS have try/catch with user-friendly toast messages on failure
- [x] **ERR-02**: Network timeout errors show "Check your connection" with retry button
- [x] **ERR-03**: Skill execution failures display the error reason and suggest specific recovery steps
- [x] **ERR-04**: Missing API key errors link directly to the Settings page with the relevant key highlighted
- [x] **ERR-05**: 404 and 500 pages use error_v2.html with contextual recovery actions
- [x] **ERR-06**: Form submission errors preserve user input (don't clear the form)

### Loading & Empty States (UX)
- [x] **UX-01**: Skill catalog shows skeleton cards while loading
- [x] **UX-02**: Execution history shows empty state with "Run your first skill" CTA when no history exists
- [x] **UX-03**: Client list shows empty state with "Add your first client" CTA when no clients exist
- [x] **UX-04**: All buttons show loading spinner during async operations and disable to prevent double-clicks
- [x] **UX-05**: Dashboard recent activity shows "No activity yet" with helpful guidance when empty
- [x] **UX-06**: Search results show "No skills match -- try different keywords" when empty

### Help & Guidance (HELP)
- [x] **HELP-01**: Each skill execution form field has a tooltip explaining what it does and example values
- [x] **HELP-02**: Settings API key section has "What is this?" and "Where do I get this?" expandable help per key
- [x] **HELP-03**: Onboarding wizard has progress indicator showing current step and total steps clearly
- [x] **HELP-04**: Help page has searchable FAQ covering the top 10 user questions
- [x] **HELP-05**: First-time dashboard visit shows welcome banner with quick orientation

### Accessibility (A11Y)
- [x] **A11Y-01**: All interactive elements have proper ARIA labels and roles
- [x] **A11Y-02**: Focus management works correctly through form flows and modals
- [x] **A11Y-03**: Color contrast meets WCAG AA standard in both dark and light themes
- [x] **A11Y-04**: All forms are keyboard-navigable (tab order, enter to submit, escape to cancel)

### Mobile Polish (MOB)
- [x] **MOB-01**: Dashboard cards stack properly on mobile (single column below 768px)
- [x] **MOB-02**: Sidebar collapses to hamburger menu on mobile
- [x] **MOB-03**: Skill execution forms are fully usable on mobile (no horizontal scroll)
- [x] **MOB-04**: Touch targets are minimum 44x44px for all buttons and links

### Workflow Streamlining (FLOW)
- [x] **FLOW-01**: "Run Again" button on output page pre-fills the form with previous parameters
- [x] **FLOW-02**: Favorite skills persist and appear on dashboard home for quick access
- [x] **FLOW-03**: Skill categories on dashboard link directly to filtered catalog view
- [x] **FLOW-04**: After onboarding completion, user lands on dashboard with first-run guidance
- [x] **FLOW-05**: Client selector dropdown available on skill execution page for client-specific runs

### Skill Discovery (DISC)
- [x] **DISC-01**: Search supports partial matches and common synonyms (e.g., "email" matches "cold-email-campaign")
- [x] **DISC-02**: Each skill card shows estimated run time and complexity indicator
- [x] **DISC-03**: Popular/recommended skills highlighted on dashboard based on user's role
- [x] **DISC-04**: Skill detail page shows example output preview or description of what gets generated

### Testing & Stability (TEST)
- [x] **TEST-01**: All existing tests continue to pass (regression)
- [x] **TEST-02**: New API endpoints have basic validation tests
- [x] **TEST-03**: End-to-end smoke test for: onboarding -> API key -> run skill -> view output
- [x] **TEST-04**: All pages render without JS errors in browser console

## v2 Requirements (Deferred)

- Batch skill execution (run multiple skills at once)
- Skill scheduling (run skills on a cron)
- Output sharing (public links to outputs)
- User analytics (usage stats, popular skills)
- Multi-language support
- Dark/light theme auto-detection from OS preference
- Keyboard shortcuts for power users

## Out of Scope

- React/Vue migration -- vanilla JS stack is sufficient and simpler to maintain
- PostgreSQL migration -- SQLite handles the load for single-user/small-team
- New skill creation -- 133 skills cover the use cases
- CI/CD pipeline -- manual Railway deploys are fine for now
- Multi-tenant auth -- current single-user login is appropriate

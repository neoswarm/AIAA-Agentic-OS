<task type="auto">
  <name>Task 1: Fix require_auth session key and wire favorites API</name>
  <files>railway_apps/aiaa_dashboard/routes/api.py</files>
  <action>
  Three changes to api.py:

  1. **Fix require_auth session key (line 38):**
     Change `session.get('authenticated')` to `session.get('logged_in')`.
     This is the one-line fix that makes all /api/* routes work for dashboard-authenticated users.
     The login handler in views.py sets `session['logged_in'] = True`, and api_v2.py already checks `session.get('logged_in')` -- api.py must match.

  2. **Add `import models` near the top:**
     After the `sys.path.insert(0, ...)` line (line 15), add `import models`.
     This is required for the favorites endpoints to call models.add_favorite(), models.remove_favorite(), etc.
     Follow the same pattern as api_v2.py line 17.

  3. **Replace the favorites stub (lines 416-450) with two working endpoints:**

     a) **Toggle endpoint** -- Replace the existing stub `api_toggle_favorite()` function entirely:
        - Route: `@api_bp.route('/favorites/toggle', methods=['POST'])`
        - Decorator: `@require_auth('write')` (works now after fix #1)
        - Parse JSON body, extract `workflow_name` (strip whitespace)
        - If empty, return 400 with `{"status": "error", "message": "workflow_name required"}`
        - Check `models.is_favorite(workflow_name)`:
          - If True: call `models.remove_favorite(workflow_name)`, return `{"status": "ok", "favorite": False}`
          - If False: call `models.add_favorite(workflow_name)`, return `{"status": "ok", "favorite": True}`
        - Wrap in try/except, return 500 on error
        - CRITICAL: Response must include `favorite` key (not `is_favorite`) -- favorites.js line 22 reads `result.favorite`

     b) **List endpoint** -- Add new function after the toggle:
        - Route: `@api_bp.route('/favorites', methods=['GET'])`
        - Decorator: `@require_auth('read')`
        - Call `models.get_favorites()` which returns `List[str]`
        - Return `jsonify(favorites)` -- a plain JSON array, NOT wrapped in an object
        - CRITICAL: favorites.js line 54 expects a bare array and iterates with `favorites.forEach(workflowName => ...)`
        - Wrap in try/except, return 500 on error

  Place the new endpoints in the same location as the old stub (between the deployment routes and the health check).
  </action>

  <name>Task 2: Add Google Docs delivery endpoint to api_v2.py</name>
  <files>railway_apps/aiaa_dashboard/routes/api_v2.py</files>
  <action>
  Add a new endpoint to api_v2.py for Google Docs delivery:

  **Route:** `@api_v2_bp.route('/executions/<execution_id>/deliver/gdocs', methods=['POST'])`
  **Decorator:** `@login_required`

  Implementation:
  1. Look up the execution via `get_execution_status(execution_id)` (already imported)
  2. If None, return 404 with `{"status": "error", "message": "Execution not found"}`
  3. Get `output_path` from the execution dict
  4. If no output_path, return 400 with `{"status": "error", "message": "No output file for this execution"}`
  5. Resolve the path: if not absolute, prepend `SKILLS_DIR.parent.parent` (the project root, matching the pattern in api_execution_output at line 260-262)
  6. If file does not exist, return 404 with `{"status": "error", "message": "Output file not found"}`
  7. Call the Google Docs delivery skill via subprocess:
     ```python
     import subprocess
     skill_script = SKILLS_DIR / "google-doc-delivery" / "create_google_doc.py"
     skill_name = execution.get("skill_name", "output")
     result = subprocess.run(
         [sys.executable, str(skill_script), "--file", str(full_path), "--title", f"{skill_name} - {execution_id[:8]}"],
         capture_output=True, text=True, timeout=60
     )
     ```
  8. If returncode != 0, return 500 with `{"status": "error", "message": "Google Docs delivery failed", "detail": result.stderr[:500]}`
  9. Parse Google Docs URL from stdout: scan lines for 'docs.google.com'
  10. Return `{"status": "ok", "message": "Delivered to Google Docs", "url": url}`
  11. Handle subprocess.TimeoutExpired with 504 status
  12. General exception handler returns 500

  Add `import subprocess` at the top of api_v2.py (near the other imports, around line 9).

  Place this endpoint in the Execution Endpoints section, after the existing `api_execution_output` function.

  CRITICAL: skill_output.html line 679 reads `data.url` to open in new tab. The response MUST include the `url` key.

  NOTE: If Google credentials are not configured on the server, the subprocess will fail and the endpoint will return a 500 with a clear error. This is the expected "degrade gracefully" pattern.
  </action>

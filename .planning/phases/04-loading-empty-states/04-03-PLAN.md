  <name>Task 1: Upgrade setButtonLoading() and add withButtonLoading() wrapper in main.js</name>
  <files>railway_apps/aiaa_dashboard/static/js/main.js</files>
  <action>
  1. Replace the existing `setButtonLoading()` function (around lines 481-496) with an improved version:

  ```javascript
  function setButtonLoading(button, loading, loadingText) {
      if (!button) return;
      if (loading) {
          // Store original state for restoration
          button.dataset.originalHTML = button.innerHTML;
          button.dataset.originalDisabled = button.disabled;
          button.disabled = true;
          var text = loadingText || button.dataset.loadingText || 'Loading...';
          button.innerHTML = '<svg width="16" height="16" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2.5" style="animation: spin 0.8s linear infinite; margin-right: 0.5rem; flex-shrink: 0;"><circle cx="12" cy="12" r="10" stroke-dasharray="32" stroke-dashoffset="12"/></svg>' + text;
      } else {
          button.disabled = button.dataset.originalDisabled === 'true';
          button.innerHTML = button.dataset.originalHTML || button.textContent;
          delete button.dataset.originalHTML;
          delete button.dataset.originalDisabled;
      }
  }
  ```

  Key changes from the old version:
  - Uses innerHTML instead of textContent for restoration (preserves SVG icons in buttons)
  - Stores originalHTML not just originalText
  - The spinner SVG uses a dashed circle (`stroke-dasharray="32" stroke-dashoffset="12"`) which creates a proper spinner look with the existing `@keyframes spin` (already in skill_execute.html CSS). The animation is inline so it works everywhere.
  - Respects `data-loading-text` attribute on buttons as fallback for custom loading text
  - Stores original disabled state so buttons that were already disabled don't become enabled after loading

  2. Add a new convenience function `withButtonLoading()` right after setButtonLoading():

  ```javascript
  async function withButtonLoading(button, asyncFn, loadingText) {
      if (!button || button.disabled) return;
      setButtonLoading(button, true, loadingText);
      try {
          return await asyncFn();
      } finally {
          setButtonLoading(button, false);
      }
  }
  ```

  This function wraps any async operation with automatic button loading state management. It prevents double-clicks (checks button.disabled), handles errors (finally block), and is the recommended pattern for new code.

  3. Add `@keyframes spin` to main.js as an injected style if not already present (for pages that don't include skill_execute.html). Add this at the top of main.js after the comment header:

  ```javascript
  // Inject spinner keyframes if not already present
  (function() {
      if (!document.getElementById('aiaa-spinner-styles')) {
          var style = document.createElement('style');
          style.id = 'aiaa-spinner-styles';
          style.textContent = '@keyframes spin { to { transform: rotate(360deg); } }';
          document.head.appendChild(style);
      }
  })();
  ```

  4. Export the new function at the bottom of main.js alongside existing exports:
  ```javascript
  window.withButtonLoading = withButtonLoading;
  ```

  Do NOT modify any other existing functions. Only replace setButtonLoading(), add withButtonLoading(), add the style injection, and add the export.

In the `createFieldElement(input)` function (around line 655), enhance the tooltip to include example values:

2. Build a richer tooltip string:
   - Start with the description text
   - If `input.placeholder` or `input.example` exists, append a line break and "Example: {value}"
   - Format: `"{description}\n\nExample: {placeholder_or_example}"`

3. Change the tooltip creation logic in createFieldElement to:
   ```javascript
   var tipText = input.description || '';
   if (input.example) {
       tipText += (tipText ? '\n\nExample: ' : 'Example: ') + input.example;
   } else if (input.placeholder && input.placeholder.length < 60) {
       tipText += (tipText ? '\n\nExample: ' : 'Example: ') + input.placeholder;
   }
   if (tipText) {
       label.innerHTML += ' <span class="field-tooltip" data-tip="' + escapeAttr(tipText) + '">?</span>';
   }
   ```

4. Update the `.field-tooltip:hover::after` CSS to handle the newline in the tooltip text. Add `white-space: pre-line;` to the existing CSS rule (around line 142-160). The rule already has `white-space: normal` which should work, but change it to `pre-line` so the `\n\n` renders as line breaks.

5. Also ensure that fields WITHOUT a description but WITH a placeholder/example still get a tooltip. The current code only adds tooltip when `input.description` exists -- the new logic should add a tooltip whenever there is ANY helpful text (description, example, or placeholder).

Do NOT change any other tooltip styling or behavior. The `?` icon, positioning, and animation should remain identical.

**E2E Smoke Test (`test_e2e_smoke`):**
A single test function that uses its own `app.test_client()` instance (NOT the auth_client fixture) to walk through the complete user journey sequentially:

Step 1 - Login:
```python
resp = client.post("/login", data={"username": "testadmin", "password": "testpass123"}, follow_redirects=False)
assert resp.status_code == 302  # Redirect on success
```

Step 2 - Save an API key:
```python
resp = client.post("/api/v2/settings/api-keys", json={"key_name": "openrouter", "key_value": "sk-or-test-1234567890abcdef"})
assert resp.status_code == 200
```

Step 3 - Browse skills:
```python
resp = client.get("/api/v2/skills")
data = resp.get_json()
assert data["status"] == "ok"
assert data["total"] > 0
skill_name = data["skills"][0]["name"]  # Pick first skill for execution
```

Step 4 - View skill detail:
```python
resp = client.get(f"/api/v2/skills/{skill_name}")
data = resp.get_json()
assert data["status"] == "ok"
```

Step 5 - Execute a skill:
Post to `/api/v2/skills/{skill_name}/execute` with json={"params": {}}. The skill may have required params -- read the skill detail from Step 4 to find required_inputs and supply dummy values. If the endpoint returns 202, assert execution_id exists. If it returns 400 due to missing required params, that ALSO proves the validation works -- assert the structured error shape and consider the test passed (the point is verifying the HTTP layer, not running actual skill scripts).

Step 6 - Check executions list:
```python
resp = client.get("/api/v2/executions")
assert resp.status_code == 200
```

Step 7 - Check API key status:
```python
resp = client.get("/api/v2/settings/api-keys/status")
assert resp.status_code == 200
```

**Important:** The E2E test verifies the HTTP request/response layer. Do NOT wait for subprocess completion from skill execution. Just verify the 202 response (or 400 validation error) is correct.

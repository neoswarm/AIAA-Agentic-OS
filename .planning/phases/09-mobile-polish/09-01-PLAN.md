3. **Add toggleMobileNav JS** as a new `<script>` block BEFORE the existing theme toggle script (before line 149). This ensures the function is defined before the DOM references it:
```javascript
function toggleMobileNav() {
    var sidebar = document.querySelector('.sidebar');
    var backdrop = document.querySelector('.sidebar-backdrop');
    var btn = document.querySelector('.hamburger-btn');
    var isOpen = sidebar.classList.contains('open');

    sidebar.classList.toggle('open');
    backdrop.classList.toggle('visible');
    btn.setAttribute('aria-expanded', String(!isOpen));
    document.body.style.overflow = isOpen ? '' : 'hidden';

    if (!isOpen) {
        var firstLink = sidebar.querySelector('.nav-item');
        if (firstLink) firstLink.focus();
    }
}

document.addEventListener('keydown', function(e) {
    if (e.key === 'Escape') {
        var sidebar = document.querySelector('.sidebar');
        if (sidebar && sidebar.classList.contains('open')) {
            toggleMobileNav();
            document.querySelector('.hamburger-btn').focus();
        }
    }
});
```

Key details:
- Use `var` not `const/let` for consistency with existing base.html scripts
- Body scroll lock (`overflow: hidden`) prevents background scrolling when sidebar is open
- Focus moves to first nav item when opening, back to hamburger when closing via Escape
- aria-expanded toggles for screen readers

2. **Replace the existing `@media (max-width: 768px)` block** (lines 938-956) with this expanded version:

```css
/* Responsive */
@media (max-width: 768px) {
    /* Hamburger visible on mobile */
    .hamburger-btn {
        display: flex;
    }

    /* Sidebar: slide-in overlay instead of display:none */
    .sidebar {
        transform: translateX(-100%);
        transition: transform 0.3s ease;
    }

    .sidebar.open {
        transform: translateX(0);
    }

    /* Main content: no sidebar margin, space for hamburger */
    .main {
        margin-left: 0;
        padding: 1rem;
        padding-top: 4rem;
    }

    /* Page titles smaller on mobile */
    .page-title {
        font-size: 1.25rem;
    }

    /* Stats grid: single column on mobile */
    .stats-grid {
        grid-template-columns: 1fr;
    }

    /* Modal: nearly full-width on mobile */
    .modal {
        width: 95%;
        max-height: 95vh;
    }

    /* Toast: full-width at bottom on mobile */
    .toast-notification {
        left: 1rem;
        right: 1rem;
        bottom: 1rem;
        min-width: auto;
    }

    .toast-container {
        left: 1rem;
        right: 1rem;
        bottom: 1rem;
        max-width: none;
    }

    /* Form buttons stack vertically on mobile */
    .form-actions {
        flex-direction: column;
    }

    .form-actions .btn-run,
    .form-actions .btn-estimate {
        width: 100%;
        justify-content: center;
    }

    /* Data tables: horizontal scroll on mobile */
    .data-table {
        display: block;
        overflow-x: auto;
    }
}
```

Key details:
- The existing `display: none` on `.sidebar` is REPLACED with `transform: translateX(-100%)` -- this is the critical fix that makes navigation accessible on mobile
- Sidebar keeps its existing `position: fixed`, `width`, and `z-index: 100` from the base styles -- only the transform changes
- `padding-top: 4rem` on `.main` prevents the hamburger button from overlapping page content
- Keep `page-title` font-size and `modal` width rules from the original block
- Add toast full-width, form button stacking, and data table horizontal scroll
- Do NOT duplicate or conflict with existing template-specific breakpoints (600px, 900px in dashboard_v2.html, skill_execute.html, etc.)

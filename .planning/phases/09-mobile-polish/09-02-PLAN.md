<task type="auto">
  <name>Task 1: Touch target enforcement (44px minimum)</name>
  <files>railway_apps/aiaa_dashboard/static/css/main.css</files>
  <action>
Append touch target rules INSIDE the existing `@media (max-width: 768px)` block in main.css (the one Plan 01 created/expanded). Add these rules after the existing mobile rules, with a clear comment separator.

Add the following INSIDE the 768px media query (before the closing `}`):

```css
    /* ---- Touch Targets: 44px minimum ---- */

    /* Navigation items */
    .nav-item {
        min-height: 44px;
        padding-top: 0.75rem;
        padding-bottom: 0.75rem;
    }

    /* Buttons, tabs, chips */
    .btn,
    .btn-secondary,
    .btn-run,
    .btn-estimate,
    .mode-tab,
    .settings-tab,
    .tab,
    .theme-toggle,
    .logout-btn,
    .category-chip,
    .category-pill {
        min-height: 44px;
    }

    /* Icon buttons and close buttons -- need both width and height */
    .modal-close,
    .toast-close,
    .toast__close,
    .qs-star,
    .welcome-banner-close,
    .search-bar__clear {
        min-width: 44px;
        min-height: 44px;
    }
```

Key details:
- Use `min-height` not `height` to avoid breaking elements that are already taller
- Nav items get explicit padding increase (was 0.625rem, now 0.75rem) to meet 44px
- Icon buttons get both min-width AND min-height since they are square targets
- Do NOT change font-size for touch targets -- use padding/min-height only
- These selectors cover elements across ALL templates (base.html sidebar, skill catalog, settings, modals, etc.)
  </action>

  <done>All buttons, nav items, tabs, chips, and icon buttons have minimum 44px touch target on mobile viewports.</done>

<task type="auto">
  <name>Task 2: iOS zoom prevention (16px form input font-size)</name>
  <files>railway_apps/aiaa_dashboard/static/css/main.css</files>
  <action>
Append iOS zoom prevention rules INSIDE the same `@media (max-width: 768px)` block, after the touch target rules from Task 1.

Add the following INSIDE the 768px media query (before the closing `}`):

```css
    /* ---- iOS Zoom Prevention: 16px form inputs ---- */
    /* iOS Safari auto-zooms viewport when focusing inputs with font-size < 16px.
       Current codebase uses 0.9375rem (15px) and 0.875rem (14px) on form fields. */

    .form-input,
    .form-select,
    .form-textarea,
    .field-input,
    .field-select,
    .field-textarea,
    .nl-textarea,
    .cron-select,
    .cron-input,
    .cron-input-full,
    .search-bar__input,
    .search-hero-input input {
        font-size: 16px;
        min-height: 44px;
    }
```

Key details:
- `font-size: 16px` is the EXACT threshold -- do not use rem, use px to ensure iOS honors it
- These selectors cover ALL input types across all templates: skill execute forms (.field-*), settings forms (.form-*), cron editor (.cron-*), search bars, and the natural language textarea
- `min-height: 44px` is added here too since form inputs are also touch targets
- This must be a separate rule from the touch target buttons rule because it includes `font-size`
- `.search-hero-input input` uses descendant selector because the input is nested inside a wrapper div
  </action>
  <verify>

2. If testing on iOS device or simulator: tap into any input field on the skill execution page -- the viewport should NOT zoom in.
3. Inspect .cron-input -- font-size should be 16px (was 14px / 0.875rem).
  </verify>
  <done>All form inputs render at 16px font-size on mobile, preventing iOS Safari auto-zoom. All form inputs also meet 44px touch target minimum.</done>

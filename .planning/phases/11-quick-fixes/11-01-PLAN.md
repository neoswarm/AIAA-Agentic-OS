<task type="auto">
  <name>Task 1: Fix onboarding API key payload, response check, and dead /clients link</name>
  <files>railway_apps/aiaa_dashboard/templates/onboarding.html</files>
  <action>
    Three changes in onboarding.html:

    1. Line 721 — Fix API key save payload field name:
       Change: `body: JSON.stringify({ key_name: 'openrouter', value: key })`
       To:     `body: JSON.stringify({ key_name: 'openrouter', key_value: key })`
       Why: The POST /api/v2/settings/api-keys endpoint reads `data.get('key_value')` (see api_v2.py line 365). Sending `value` means the API receives an empty key_value and returns a validation error.

    2. Line 727 — Fix success response detection:
       Change: `if (data.success) {`
       To:     `if (data.status === 'ok') {`
       Why: The API returns `{"status": "ok", "message": "..."}` on success (see api_v2.py line 394-397). Checking `data.success` is always falsy since that field does not exist in the response.

    3. Line 646 — Fix dead /clients navigation link:
       Change: `<a href="/clients" class="next-step-link">`
       To:     `<a href="/clients-manage" class="next-step-link">`
       Why: The route is registered as `/clients-manage` in views.py line 707. `/clients` returns 404.
  </action>
  <verify>
    grep -n 'key_value: key' railway_apps/aiaa_dashboard/templates/onboarding.html | grep -q '721' || grep -n 'key_value' railway_apps/aiaa_dashboard/templates/onboarding.html
    grep -n "data.status === 'ok'" railway_apps/aiaa_dashboard/templates/onboarding.html
    grep -n '/clients-manage' railway_apps/aiaa_dashboard/templates/onboarding.html | grep -q 'next-step-link'
    Confirm no remaining instances of `value: key` (wrong field), `data.success` (wrong check), or `href="/clients"` (wrong route) in onboarding.html.
  </verify>
  <done>
    - Line ~721 sends `key_value: key` in the POST body
    - Line ~727 checks `data.status === 'ok'` for success
    - Line ~646 links to `/clients-manage`
    - No other instances of the old broken patterns remain in onboarding.html
  </done>

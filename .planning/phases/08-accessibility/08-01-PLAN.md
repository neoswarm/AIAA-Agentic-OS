**base.html changes:**
1. Add skip-to-content link as first child of `<body>`:
   `<a href="#main-content" class="skip-link">Skip to main content</a>`
   (CSS for skip-link will be added in Plan 08-02; for now just add the HTML)

2. Add `id="main-content"` and `tabindex="-1"` to the `<main>` element

3. Add `aria-label="Main navigation"` to the `<nav>` element inside `<aside>`

4. Add `aria-label="Sidebar"` to the `<aside>` element

5. Convert theme toggle from `<div class="theme-toggle" onclick="toggleTheme()">` to `<button class="theme-toggle" onclick="toggleTheme()" aria-label="Toggle dark and light theme">`. Keep all inner content. Remove any div-specific styling issues.

6. Convert advanced nav toggle from `<div class="nav-label-advanced" onclick="...">` to `<button class="nav-label-advanced" onclick="..." aria-expanded="false" aria-label="Toggle advanced navigation">`. Update the onclick JS to toggle `aria-expanded` attribute.

7. Add `aria-live="polite" aria-atomic="true"` to `#toast-container`

8. Add `aria-hidden="true"` to ALL decorative `<svg>` elements in the sidebar navigation (these are paired with text labels so purely decorative)

**main.js changes:**
9. Add `trapFocus(modalElement)` utility function at the module level (see 08-RESEARCH.md Pattern 1 for exact implementation). It should:
   - Query all focusable elements inside modalElement
   - Trap Tab/Shift+Tab within those elements
   - Fire 'modal-escape' CustomEvent on Escape key
   - Focus the first focusable element
   - Return a cleanup function

10. Update `confirmAction(message)` to be accessible:
    - Store `document.activeElement` as triggerElement before creating modal
    - Add `role="dialog"`, `aria-modal="true"`, `aria-labelledby="confirm-title"`, `aria-describedby="confirm-message"` to the modal div
    - Add `id="confirm-title"` to the title and `id="confirm-message"` to the message paragraph
    - Call `trapFocus()` on modal open
    - Call cleanup and `triggerElement.focus()` on modal close (both confirm and cancel paths)
    - Listen for 'modal-escape' event to close with false
    - See 08-RESEARCH.md Example 1 for full implementation

11. Update `showToast()` to set `role="alert"` for error toasts and `role="status"` for info/success toasts on each toast element

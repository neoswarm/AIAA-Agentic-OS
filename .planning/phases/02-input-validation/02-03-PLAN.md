**2. Skill catalog search (skill_execution.js, line ~587-589):**

The existing `debouncedSkillSearch` uses `debounce(function(event) { searchSkills(event.target.value); }, 300)` -- already at 300ms. Good.

However, inside `searchSkills()` (line ~15), the query is sent to `/api/v2/skills/search?q=...` via `fetchAPI`. The query is already URL-encoded via `encodeURIComponent()` which handles XSS for the request. But the response rendering (line ~31-33) uses `escapeHtml(query)` for the "No skills found" message, and `escapeHtml(skill.display_name)` / `escapeHtml(skill.description)` for results. Good -- XSS is handled.

Add whitespace normalization: in `searchSkills()`, after the `query.trim()` check, add `query = query.trim().replace(/\s+/g, ' ');` before sending to API.

**3. Client search (clients.html, line ~412-419):**

The current implementation does NOT use debounce:
```javascript
document.getElementById('clientSearchInput').addEventListener('input', function() {
    var query = this.value.toLowerCase();
    var rows = document.querySelectorAll('#clientTableBody tr');
    // ...
});
```

This is a local DOM filter (no API calls), so the excessive API call concern doesn't apply. However:
- Add debounce for smoother UX on large client lists: wrap in debounce with 200ms (shorter since it's local)
- Add whitespace trimming: `var query = this.value.trim().toLowerCase();`
- Verify that the client table rendering uses `escapeHtml()` -- check `renderClients()` (line ~554). It uses `escapeHtml()` for name, industry, website, and slug. Good.

Update to:
```javascript
var debouncedClientSearch = (function() {
    var timeout;
    return function(value) {
        clearTimeout(timeout);
        timeout = setTimeout(function() {
            var query = value.trim().toLowerCase().replace(/\s+/g, ' ');
            var rows = document.querySelectorAll('#clientTableBody tr');
            rows.forEach(function(row) {
                var text = row.textContent.toLowerCase();
                row.style.display = text.includes(query) ? '' : 'none';
            });
        }, 200);
    };
})();

document.getElementById('clientSearchInput').addEventListener('input', function() {
    debouncedClientSearch(this.value);
});
```

Note: Use an IIFE for the debounce since the main.js `debounce()` utility might not be loaded on this page (clients.html doesn't include main.js explicitly -- it's only in base.html... actually check base.html). Looking at base.html, there's no `<script src="main.js">` inclusion. The main.js functions (showToast, debounce, etc.) are available because base.html defines `showToast()` inline, but NOT `debounce()`. The skill_execution.js file calls `debounce()` from main.js, so main.js must be loaded somewhere. Check if it's loaded via v2.css or another mechanism. If main.js is not reliably available on all pages, use the IIFE pattern.
